DPDP Scanner - Unified Setup + Packaging

1) Build the scanner package
Use `bootstrap_and_package.py` with `automation_runner_config.json`.

From project root:
python3 bootstrap_and_package.py --config automation_runner_config.json

Expected build outputs:
- `dist/dpdp-pii-scanner/` (packaged folder)
- `dist/dpdp-pii-scanner-<os>-<arch>.zip` (bundle)

2) Run the scanner after successful build (recommended)
Run from the project root using the launcher `./dpdp-scan`.

macOS/Linux:
./dpdp-scan --help

Windows:
cd dist\dpdp-pii-scanner
dpdp-scan.cmd --help

Run with packaged default config:
- macOS/Linux (project root): `./dpdp-scan`
- Windows: `dpdp-scan.cmd`

Run with explicit config:
- macOS/Linux: `./dpdp-scan --config ./config/scanner/scanner_config.json`
- Windows: `dpdp-scan.cmd --config C:\full\path\config\scanner\scanner_config.json`

Scanner configs live in `config/scanner/` (for example `scanner_config.json`,
`run_scanner_mysql_and_files.json`, `run_scanner_mysql_postgres_and_files.json`,
`run_scanner_mysql_local.json`, `run_scanner_postgres_local.json`).

Common runtime overrides:
- Scan specific paths (repeatable):
  `./dpdp-scan --path /data/files --path /extra/share`
- Change output file:
  `./dpdp-scan --output output/final_output.json`
- Switch rules environment:
  `./dpdp-scan --rules-env prod`
- Mask file paths in output:
  `./dpdp-scan --mask-file-paths --file-path-mask-mode hash`
- Enable database scanning with a profile:
  `./dpdp-scan --enable-db --db mysql_local`
- Override database profile directory:
  `./dpdp-scan --enable-db --db mysql_local --db-config-dir /path/to/db_profiles`

DPDP_MYSQL_PASSWORD='' ./dist/dpdp-pii-scanner/dpdp-scan \
  --config config/scanner/run_scanner_mysql_and_files.json \
  --output output/output.json

DPDP_MYSQL_PASSWORD='' DPDP_POSTGRES_PASSWORD='dpdp_scanner' ./dist/dpdp-pii-scanner/dpdp-scan \
  --config config/scanner/run_scanner_mysql_postgres_and_files.json \
  --output output/output.json

DB-only runs with local sample configs:
DPDP_MYSQL_PASSWORD='' ./dist/dpdp-pii-scanner/dpdp-scan \
  --config config/scanner/run_scanner_mysql_local.json

DPDP_POSTGRES_PASSWORD='dpdp_scanner' ./dist/dpdp-pii-scanner/dpdp-scan \
  --config config/scanner/run_scanner_postgres_local.json

Local sample DB setup helper (optional install + seed):
./test_data/database/setup_local_databases.sh --install

3) Run from source (alternative to packaged binary)
Use the isolated virtual environment:

macOS/Linux:
source .venv/bin/activate
python main.py --config config/scanner/scanner_config.json

Windows:
.venv\Scripts\activate
python main.py --config config/scanner/scanner_config.json

4) All available scanner CLI options
`main.py` / packaged `dpdp-scan` support:

- `-h, --help`
- `--config CONFIG`
- `--init-config PATH`
- `--path PATHS` (repeatable)
- `--output OUTPUT`
- `--log-level LOG_LEVEL` (`DEBUG|INFO|WARNING|ERROR`)
- `--rules-env RULES_ENV` (for example `default|dev|qa|prod`)
- `--mask-file-paths`
- `--file-path-mask-mode {full,basename,relative,hash,redacted}`
- `--enable-db`
- `--db, --db-profile` (repeatable; profile name or JSON path)
- `--db-config-dir`

6) Runner config (`automation_runner_config.json`) summary
Main keys:
- `target_environment`: `auto|mac|linux|windows`
- `python`
- `environment`
- `dependencies`
- `database_test_data`
- `packaging`
- `runtime_smoke_test`

Key behavior:
- Keep `environment.clear=false` to reuse one isolated `.venv`.
- Keep `environment.preserve_existing_venv_when_offline=true` for offline reuse.
- `packaging.remove_incompatible_packages=true` removes known PyInstaller blockers
  like `pathlib` before packaging.

7) Notes
- DB scanning uses `https://github.com/tokern/piicatcher` only.
- Database profiles live under `config/databases`; load them with `--db` or
  `sources.database.profiles`.
- For RDS, configure connection `type` and `url` according to engine.
- If a dbcat source method name differs in your environment, use
  `add_source_function_candidates` at global or connection level.
- First-time bootstrap requires internet access.
- Offline runs work only with a pre-populated `.venv` and `environment.clear=false`.
